Flexbox

What is Flexbox?

Flexbox provides a more efficient way to lay out,
align and distribute space among items in a container,
even when their size is unknown/dynamic

Overview

The job of a flex container is to group a bunch of flex items together and define how they’re positioned

History and Basics

  • Developed alternative to Floats to break out its limitations
  • Well designed for small sized applications

Floats vs Flexbox

How to start using it?

Simple.
By turning any of our HTML elements into a flex container

Then, all children of the container become flex items.

 display: flex

Layout

The CSS flexbox has two axes: the main and the cross

flex-direction

You define main axis with flex direction property

 flex-direction: row | row-reverse | column | column-reverse;

flex-wrap

By default, flex items will all try to fit onto one line

It determines whether overflow should be wrapped or not

 flex-wrap: nowrap | wrap | wrap-reverse;

justify-content

justify-content property sets how browsers handle space surrounding and between elements along the main axis

 justify-contet: flex-start | flex-end | center | space-between | space-around | space-evenly | start | end | left | right

align-items

This defines the default behavior for how flex items are laid out along the cross axis on the current line

Think of it as the justify-content version for the cross axis (perpendicular to the main-axis) 

 align-items: stretch | flex-start | flex-end | center | baseline | first baseline | last baseline | start | end | self-start | self-end

align-items

align-content

It is similar to align-items, but it aligns lines of items within a container. It works along the cross axis

 align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch | start | end | baseline | first baseline | last baseline

order

By default, flex items in the container appear in the same order in which they come in code (source order)

You can change this sequence with the order property


            .item {
                order: integer; /* default is 0 */
            }

flex-grow

It defines how much a flex item expands in relation to other items (if there is available space for that)

Negative numbers don’t work for this property


            .item {
                flex-grow: number; /* default 0 */
              }

flex-shrink

This defines the ability for a flex item to shrink if necessary

Negative numbers don’t work for this property


            .item {
                flex-shrink: number; /* default 1 */
              }

flex-basis

The flex-basis property defines the initial size of a flex item. It accepts length units or percentages

Negative numbers don’t work for this property


            .item {
              flex-basis: length | auto; /* default auto */
              }

flex

This is the shorthand for flex-grow, flex-shrink and flex-basis combined.

flex-shrink and flex-basis are optional. Default is 0 1 auto


            .item {
              flex: none | flex-grow flex-shrink ? || flex-basis
            }

align-self

This allows the default alignment (or the one specified by align-items) to be overridden for individual flex items


            .item {
              align-self: auto | flex-start | flex-end | center | baseline | stretch;
            }

Note

  • float
  • vertical-align
  • clear

have no effect on a flex item
