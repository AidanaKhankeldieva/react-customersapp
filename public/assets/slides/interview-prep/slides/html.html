4 - Custom attributes

data-*

Why?

to store custom data to the page or application

Have you stored data inside class or id attribute so you can use in javascript?

[{ id:5, name:"Seytech" }]

<div id="5"> Seytech </div>

We could use like this?

[{ id:5, name:"Seytech", category: 'education' }]

<div data-id="5" data-category="education"> Seytech </div>

data-*

The data attribute name must be at least one character long and must be prefixed with 'data-'. It should not contain any uppercase letters.

How to use from js?

elem.hasAttribute(name) – checks for existence.
elem.getAttribute(name) – gets the value.
elem.setAttribute(name, value) – sets the value.
elem.removeAttribute(name) – removes the attribute.

5 - Block vs Inline Elements

A block-level element

always starts on a new line and takes up the full width available

An inline-level element

An inline element does not start on a new line and only takes up as much width as necessary.

Block elements

<div>, <h1-h6>, <header>, <footer>, <ul>, <li>, <ol>, <table> etc

Inline elements

<span>, <a>, <button>, <img>, <input>, <strong>, <script> etc

Div vs Span

  • <div> usually used as a container for other HTML elements and to separate them for the rest.
  • <span> usually used as a container for HTML text. But essentially, it is used to style a certain text within a larger text element.

  • semantic html - helps screen readers
  • declare language - Indicate the primary language of every page
